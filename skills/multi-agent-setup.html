🏗️ 多Agent团队配置

一只龙虾变一支团队——让你的Bot学会配置和管理多个AI角色协作

← 三万同款

💡 真实故事：傅盛的龙虾三万，14天从1只裂变成8个Agent团队：总指挥接收任务→笔杆子写文章→参谋做研究→运营官管日常→进化官写代码→社区官管Discord→交易官盯股票。7×24小时自动协作，老板只需要在飞书说一句话。

🏛️ 架构概览

多Agent = 一个OpenClaw实例 + 多个独立角色，各有人设、记忆、技能

🎯 总指挥（主Agent）
↙ ↓ ↘
✍️ 笔杆子 🔬 参谋 📋 运营官 🧬 进化官 💬 社区官
sessions_spawn 派任务 · 共享文件协作 · sessions_send 持续对话

📋 配置步骤

1
定义Agent列表（agents.list）
在 ~/.openclaw/openclaw.json 的 agents 字段中定义所有角色：
{
  "agents": {
    "defaults": {
      "model": { "primary": "amazon-bedrock/us.anthropic.claude-sonnet-4-6" }
    },
    "list": [
      {
        "id": "main",
        "default": true,
        "name": "总指挥",
        "workspace": "/home/user/.openclaw/workspace",
        "subagents": {
          "allowAgents": ["creator", "canmou", "yunying"]
        }
      },
      {
        "id": "creator",
        "name": "笔杆子",
        "workspace": "/home/user/.openclaw/workspace-creator"
      },
      {
        "id": "canmou",
        "name": "参谋",
        "workspace": "/home/user/.openclaw/workspace-canmou"
      },
      {
        "id": "yunying",
        "name": "运营官",
        "workspace": "/home/user/.openclaw/workspace-yunying"
      }
    ]
  }
}
2
创建每个Agent的Workspace
每个Agent需要独立的工作目录和人设文件：
# 创建目录
mkdir -p ~/.openclaw/workspace-creator
mkdir -p ~/.openclaw/workspace-canmou
mkdir -p ~/.openclaw/workspace-yunying

# 每个目录下创建 SOUL.md（人设文件）
# 例：笔杆子的 SOUL.md
cat > ~/.openclaw/workspace-creator/SOUL.md << 'EOF'
# 笔杆子

## 我是谁
我是笔杆子，专职内容创作者。

## 职责
- 写公众号文章、Twitter Thread
- 内容风格把控
- 配合总指挥的任务安排

## 写作风格
- 口语化、有故事感
- 数据支撑但不堆砌
- 有态度有观点
EOF
3
绑定飞书/Discord渠道（可选）
如果要每个Agent独立的聊天入口：
// 方案A：共享一个飞书Bot（推荐新手）
// 所有消息发给主Agent，主Agent内部派发
"bindings": [
  { "agentId": "main", "match": { "channel": "feishu", "accountId": "my-app" } }
]

// 方案B：每个Agent一个飞书Bot
// 用户可以直接和特定角色对话
"bindings": [
  { "agentId": "main", "match": { "channel": "feishu", "accountId": "app-main" } },
  { "agentId": "creator", "match": { "channel": "feishu", "accountId": "app-creator" } }
]
💡 方案A最简单：只需一个飞书Bot，主Agent自动分工。方案B更强但需要额外创建飞书应用。
4
重启Gateway生效
# 修改完 openclaw.json 后重启
openclaw gateway restart

# 验证Agent列表
openclaw status
5
主Agent调度子Agent
三种协作方式：
// 方式1：派发一次性任务（最常用）
sessions_spawn(
  task="写一篇关于AI未来趋势的文章，800字",
  agentId="creator"
)
// 子Agent完成后自动回传结果

// 方式2：给已有session发消息（持续对话）
sessions_send(
  agentId="canmou",
  message="帮我分析一下这份竞品报告"
)

// 方式3：共享文件（异步协作）
write("/home/user/.openclaw/workspace/tasks/today.md", "今日待办：...")

📊 三种典型方案

方案架构适合复杂度
🟢 A：单Bot+多Agent一个飞书Bot入口，内部多角色个人用户⭐
🔵 B：多Bot+多Agent每个Agent独立飞书Bot企业内部⭐⭐⭐
🟣 C：混合渠道飞书+Discord+Telegram混合全球团队⭐⭐⭐⭐

⚠️ 重要提醒

配置前必须想清楚：
• 你需要几个角色？每个负责什么？
• 哪个是主Agent（总指挥）？
• 是否需要独立飞书Bot？（推荐先方案A）
• 模型选择：重要角色用Opus/Sonnet，轻量角色用Haiku省钱
💡 最佳实践（三万14天踩坑总结）：
• 先跑通2个Agent再加更多，不要一口气配7个
• 每个Agent的SOUL.md要写清楚职责边界
• 主Agent的AGENTS.md里写清楚派发规则
• 子Agent没有主Agent的记忆，spawn时要给完整上下文
• 定期巡检子Agent（别让他们偷懒！）
